    O cdigo 128 uma simbologia de cdigo de barras linear de alta densidade definida na ISO/IEC 15417:2007. [1] usado para cdigos de barras alfanumricos ou apenas numricos . Ele pode codificar todos os 128 caracteres ASCII e, usando um smbolo de extenso (FNC4), os caracteres Latin-1 definidos na ISO/IEC 8859-1 . [ citao necessria ] . Geralmente resulta em cdigos de barras mais compactos em comparao com outros mtodos como o Code 39 , especialmente quando os textos contm principalmente dgitos. O Code 128 foi desenvolvido pela Computer Identics Corporation em 1981. [2]

    Um cdigo de barras GS1-128 codificando GTIN, data de validade e nmero do lote

    GS1-128 (anteriormente conhecido como UCC/EAN-128) um subconjunto do Cdigo 128 e amplamente utilizado em todo o mundo nas indstrias de transporte e embalagem como um cdigo de identificao de produto para os nveis de continer e palete na cadeia de suprimentos.

    Especificao [ editar ]

    Esquema de um cdigo de barras (Cdigo 128B).
    1 :zona silenciosa, 2 :cdigo de incio, 3 :dados, 4 :soma de verificao, 5 :parar

    Um cdigo de barras Code 128 tem sete sees:

    1. Zona quieta
    2. Smbolo de incio
    3. Dados codificados
    4. Verifique o smbolo (obrigatrio)
    5. smbolo de parada
    6. Barra final (geralmente considerada parte do smbolo de parada)
    7. Zona quieta

    O smbolo de verificao calculado a partir de uma soma ponderada ( mdulo 103) de todos os smbolos.

    Subtipos [ editar ]

    O cdigo 128 inclui 108 smbolos: 103 smbolos de dados, 3 smbolos de incio e 2 smbolos de parada. Cada smbolo consiste em trs barras pretas e trs espaos em branco de larguras variadas. Todas as larguras so mltiplos de um "mdulo" bsico. Cada barra e espao tem de 1 a 4 mdulos de largura, e os smbolos so de largura fixa: a soma das larguras das trs barras pretas e das trs barras brancas 11 mdulos.

    O padro de parada composto por dois smbolos sobrepostos e possui quatro barras. O padro de parada permite a varredura bidirecional. Quando o padro de parada lido da esquerda para a direita (o caso usual), o smbolo de parada (seguido por uma barra de 2 mdulos) reconhecido. Quando o padro de parada lido da direita para a esquerda, o smbolo de parada reversa (seguido por uma barra de 2 mdulos) reconhecido. Um leitor que v o smbolo de parada reverso sabe que deve pular a barra de 2 mdulos e ler o restante do cdigo de barras ao contrrio.

    Apesar do nome, o Code 128 no possui 128 smbolos distintos, portanto no pode representar 128 pontos de cdigo diretamente. Para representar todos os 128 valores ASCII, ele alterna entre trs conjuntos de cdigos (A, B, C). Juntos, os conjuntos de cdigos A e B abrangem todos os 128 caracteres ASCII. O conjunto de cdigos C usado para codificar sequncias de dgitos com eficincia. O subconjunto inicial selecionado usando o smbolo de incio apropriado. Dentro de cada conjunto de cdigos, alguns dos 103 pontos de cdigo de dados so reservados para mudar para um dos outros dois conjuntos de cdigos. As mudanas so feitas usando os pontos de cdigo 98 e 99 nos conjuntos de cdigos A e B, 100 nos conjuntos de cdigos A e C e 101 nos conjuntos de cdigos B e C para alternar entre eles):

    • 128A (conjunto de cdigos A) caracteres ASCII de 00 a 95 (09, AZ e cdigos de controle), caracteres especiais e FNC 14
    • 128B (Conjunto de cdigos B) caracteres ASCII 32 a 127 (09, AZ, az), caracteres especiais e FNC 14
    • 128C (conjunto de cdigos C) 0099 (codifica dois dgitos com um nico ponto de cdigo) e FNC1

    A largura mnima da zona silenciosa esquerda e direita do Cdigo 128 10x, onde x a largura mnima de um mdulo. obrigatrio no lado esquerdo e direito do cdigo de barras.

    Iniciar/parar e dados codificados [ editar ]

    Cada smbolo no cdigo de barras composto por trs barras e trs espaos. Cada barra ou espao tem 1, 2, 3 ou 4 unidades de largura, a soma das larguras das barras deve ser par (4, 6 ou 8 unidades), a soma das larguras dos espaos deve ser mpar (3, 5 ou 7 unidades), totalizando 11 unidades por smbolo. Por exemplo, codificar o caractere ASCII "0" pode ser visto como 10011101100, onde uma sequncia de 1's uma barra e uma sequncia de 0's um espao. Um nico 1 seria a linha mais fina no cdigo de barras. Trs 1's em sequncia (111) indicam uma barra trs vezes mais espessa que uma nica barra 1.

    Existem 108 smbolos possveis de 11 unidades de largura, e o cdigo usa todos os smbolos possveis. Dois dos smbolos so usados para indicao de parada (fim do cdigo de barras), parada e parada reversa. Os dois smbolos de parada so especiais porque so sempre seguidos por uma barra de 2 unidades, formando um padro de parada de 13 unidades. Ler o padro de parada da esquerda para a direita o smbolo de parada (seguido por uma barra de 2 unidades) e ler o padro de parada da direita para a esquerda o smbolo de parada reversa (seguido por uma barra de 2 unidades).

    Verifique o clculo do dgito [ editar ]

    O dgito de verificao uma soma de verificao de mdulo-103 ponderada. calculado somando o 'valor' do cdigo inicial aos produtos do 'valor' de cada smbolo multiplicado por sua posio na sequncia do cdigo de barras. O smbolo inicial e o primeiro smbolo codificado esto na posio 1. A soma dos produtos ento reduzida mdulo 103. O restante ento convertido de volta para um dos 103 smbolos no delimitadores (seguindo as instrues fornecidas abaixo) e anexado ao cdigo de barras , imediatamente antes do smbolo de parada.

    Usando FNC4 para codificar caracteres altos (160255) [ editar ]

    O smbolo especial FNC4 ("Funo 4"), presente apenas nos conjuntos de cdigos A e B, pode ser usado para codificar todos os caracteres Latin-1 ( ISO-8859-1 ) em um cdigo de barras Code 128. [3] O recurso no amplamente suportado e no usado no GS1-128. [4] [5] Quando um nico FNC4 est presente em uma string, o smbolo a seguir convertido em ASCII como de costume e, em seguida, 128 adicionado ao valor ASCII. (Se o smbolo a seguir for uma mudana, um segundo smbolo ser usado para obter o caractere.) Se dois FNC4s forem usados consecutivamente, todos os caracteres a seguir sero tratados como tal, at o final da string ou outro par de FNC4s . Entre os FNC4s duplos, um nico FNC4 ser usado para indicar que o seguinte caractere ser ASCII padro .[6]

    Larguras do cdigo de barras [ editar ]

    Code128 especifica uma combinao de 6 barras alternadas e espaos (3 de cada) para cada smbolo. Assim, cada smbolo comea com uma barra e termina com um espao. Em fontes de cdigo de barras, a barra final geralmente combinada com o smbolo de parada para criar um padro de parada mais amplo. A tabela a seguir detalha as larguras associadas a cada barra e espao para cada smbolo. A largura de cada barra ou espao pode ser de 1, 2, 3 ou 4 unidades (mdulos). Usando o exemplo acima, um 'A' seria representado com o padro 10100011000, ou como larguras 111323 nas tabelas abaixo.

    O valor das larguras derivado contando o comprimento de cada execuo de 1 e 0 no padro, comeando da esquerda. Sempre haver 6 execues e os comprimentos dessas 6 execues formam o valor de Larguras. Por exemplo, usando o padro 10100011000, os comprimentos de execuo so 1 (dgito 1), 1 (dgito 0), 1 (dgito 1), 3 (dgito 0), 2 (dgito 1), 3 (dgito 0). Relatar apenas os comprimentos de cada execuo fornece 1, 1, 1, 3, 2, 3, produzindo assim um valor de largura de 111323.

    Os smbolos "Cdigo A", "Cdigo B" e "Cdigo C" fazem com que todos os smbolos a seguir sejam interpretados de acordo com o subcdigo correspondente (ou seja, 128A, 128B ou 128C). O smbolo "Shift" alterna a interpretao de um nico smbolo seguinte entre os subcdigos A e B.

    O caractere ASCII codificado depende da fonte do cdigo de barras usada. Especialmente o caractere ASCII de valor 0 e de valor 95 e acima pode ser definido de forma diferente na fonte instalada.

    Os cdigos FNC x so usados para fins especiais. FNC1 no incio de um cdigo de barras indica um cdigo de barras GS1-128 que comea com um identificador de aplicativo de 2, 3 ou 4 dgitos atribudo pelo Uniform Code Council , que explica os dgitos a seguir. Por exemplo, o identificador de aplicativo 421 indica que segue um cdigo de pas numrico ISO 3166-1 e um cdigo postal de destino. Assim, o cdigo postal dos EUA para a Casa Branca geralmente seria impresso como "(421) 840 20500", mas na verdade seria codificado como " [Start C] [FNC1] 42 18 40 20 50 [Code A] 16 [Check symbol 92] [Stop]"

    FNC2 (message append) can appear anywhere within a barcode to indicate that the barcode reader should store the current string and prepend it to the string of the next barcode that's read. It is not used by GS1-128.

    FNC3 (initialize) can appear anywhere within a barcode to instruct the barcode reader to initialize or reprogram itself according to the barcode string's instructions.

    FNC4 is used to represent an extended ASCII character set (see  Using FNC4 to encode high (160255) characters). It is not used by GS1-128.[7]

    Availability[edit]

    For the end user, Code 128 barcodes may be generated by either an outside application to create an image of the barcode, or by a font-based barcode solution. Either solution requires the use of an application or an application add in to calculate the check digit and create the barcode.

    Barcode length optimization[edit]

    Code set C uses one code symbol to represent two digits, so when the text contains just digits it will generally result in shorter barcodes. However, when the string contains only a few digits or it's mixed with non-digit character, it does not always produce a more compact code than code sets A or B. Using code set C saves one symbol per two digits, but costs a mode-shift symbol to enter and exit the set. Thus, it is only worth using if there are enough consecutive digits. For example, encoding the string "X00Y" with code set A or B requires 7 code symbols ([Start B] 56 16 16 57 [checksum] [Stop]), while using code set C for the "X00Y" would result in a code 8 symbols long ([Start B] 56 [Code C] 00 [Code B] 57 [checksum] [Stop]).

    Using code set C is only advantageous under the following conditions:

    Location of digits Number of consecutive digits
    beginning of data 4+
    end of data 4+
    middle of data (surrounded by symbols from code set A or B) 6+
    entire data either 2 or 4+ (but not 3)

    At the end of a string, delaying the transition to code set C until there are an even number of digits remaining avoids an extra symbol. Consider the string "...01234": a delayed switch produces ... 0 [Code C] 12 34 [checksum] [Stop] but an early switch produces ... [Code C] 01 23 [Code A] 4 [checksum] [Stop].[8]

    For example, given the string "098x1234567y23", savings on barcode length using code set C are achieved only if it is applied to middle part of the string. For the beginning and ending part of the string, switching to code set C is not effective. As there are an odd number of digits in the middle of the string, the odd one must be use a different code, set, but it makes no difference whether this is the first or last; 16 symbols are required in either case: [Start B] 0 9 8 x 1 [Code C] 23 45 67 [Code B] y 2 3 [checksum] [Stop], or [Start B] 0 9 8 x [Code C] 12 34 56 [Code B] 7 y 2 3 [checksum] [Stop].

    Optimizing the length of the resulting barcode is important when barcode readers are used which must detect the entire barcode image at once in order to read it, such as common laser scanners. The longer the barcode is, the greater distance of laser barcode reader from barcode image is needed, making reading difficult or impossible above some threshold lengths/distances.

    The optimal encoding can be found using a dynamic programming algorithm.[9]
