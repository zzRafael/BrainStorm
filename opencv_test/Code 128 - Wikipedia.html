    Um código de barras postal suíço codificando "RI 476 394 652 CH" no código 128 (B & C)

    O código 128 é uma simbologia de código de barras linear de alta densidade definida na ISO/IEC 15417:2007. [1] É usado para códigos de barras alfanuméricos ou apenas numéricos . Ele pode codificar todos os 128 caracteres ASCII e, usando um símbolo de extensão (FNC4), os caracteres Latin-1 definidos na ISO/IEC 8859-1 . [ citação necessária ] . Geralmente resulta em códigos de barras mais compactos em comparação com outros métodos como o Code 39 , especialmente quando os textos contêm principalmente dígitos. O Code 128 foi desenvolvido pela Computer Identics Corporation em 1981. [2]

    Um código de barras GS1-128 codificando GTIN, data de validade e número do lote

    GS1-128 (anteriormente conhecido como UCC/EAN-128) é um subconjunto do Código 128 e é amplamente utilizado em todo o mundo nas indústrias de transporte e embalagem como um código de identificação de produto para os níveis de contêiner e palete na cadeia de suprimentos.

    Especificação [ editar ]

    Esquema de um código de barras (Código 128B).
    1 :zona silenciosa, 2 :código de início, 3 :dados, 4 :soma de verificação, 5 :parar

    Um código de barras Code 128 tem sete seções:

    1. Zona quieta
    2. Símbolo de início
    3. Dados codificados
    4. Verifique o símbolo (obrigatório)
    5. símbolo de parada
    6. Barra final (geralmente considerada parte do símbolo de parada)
    7. Zona quieta

    O símbolo de verificação é calculado a partir de uma soma ponderada ( módulo 103) de todos os símbolos.

    Subtipos [ editar ]

    O código 128 inclui 108 símbolos: 103 símbolos de dados, 3 símbolos de início e 2 símbolos de parada. Cada símbolo consiste em três barras pretas e três espaços em branco de larguras variadas. Todas as larguras são múltiplos de um "módulo" básico. Cada barra e espaço tem de 1 a 4 módulos de largura, e os símbolos são de largura fixa: a soma das larguras das três barras pretas e das três barras brancas é 11 módulos.

    O padrão de parada é composto por dois símbolos sobrepostos e possui quatro barras. O padrão de parada permite a varredura bidirecional. Quando o padrão de parada é lido da esquerda para a direita (o caso usual), o símbolo de parada (seguido por uma barra de 2 módulos) é reconhecido. Quando o padrão de parada é lido da direita para a esquerda, o símbolo de parada reversa (seguido por uma barra de 2 módulos) é reconhecido. Um leitor que vê o símbolo de parada reverso sabe que deve pular a barra de 2 módulos e ler o restante do código de barras ao contrário.

    Apesar do nome, o Code 128 não possui 128 símbolos distintos, portanto não pode representar 128 pontos de código diretamente. Para representar todos os 128 valores ASCII, ele alterna entre três conjuntos de códigos (A, B, C). Juntos, os conjuntos de códigos A e B abrangem todos os 128 caracteres ASCII. O conjunto de códigos C é usado para codificar sequências de dígitos com eficiência. O subconjunto inicial é selecionado usando o símbolo de início apropriado. Dentro de cada conjunto de códigos, alguns dos 103 pontos de código de dados são reservados para mudar para um dos outros dois conjuntos de códigos. As mudanças são feitas usando os pontos de código 98 e 99 nos conjuntos de códigos A e B, 100 nos conjuntos de códigos A e C e 101 nos conjuntos de códigos B e C para alternar entre eles):

    • 128A (conjunto de códigos A) – caracteres ASCII de 00 a 95 (0–9, A–Z e códigos de controle), caracteres especiais e FNC 1–4
    • 128B (Conjunto de códigos B) – caracteres ASCII 32 a 127 (0–9, A–Z, a–z), caracteres especiais e FNC 1–4
    • 128C (conjunto de códigos C) – 00–99 (codifica dois dígitos com um único ponto de código) e FNC1

    A largura mínima da zona silenciosa à esquerda e à direita do Código 128 é 10x, onde x é a largura mínima de um módulo. É obrigatório no lado esquerdo e direito do código de barras.

    Iniciar/parar e dados codificados [ editar ]

    Cada símbolo no código de barras é composto por três barras e três espaços. Cada barra ou espaço tem 1, 2, 3 ou 4 unidades de largura, a soma das larguras das barras deve ser par (4, 6 ou 8 unidades), a soma das larguras dos espaços deve ser ímpar (3, 5 ou 7 unidades), totalizando 11 unidades por símbolo. Por exemplo, codificar o caractere ASCII "0" pode ser visto como 10011101100, onde uma sequência de 1's é uma barra e uma sequência de 0's é um espaço. Um único 1 seria a linha mais fina no código de barras. Três 1's em sequência (111) indicam uma barra três vezes mais espessa que uma única barra 1.

    Existem 108 símbolos possíveis de 11 unidades de largura, e o código usa todos os símbolos possíveis. Dois dos símbolos são usados ​​para indicação de parada (fim do código de barras), parada e parada reversa. Os dois símbolos de parada são especiais porque são sempre seguidos por uma barra de 2 unidades, formando um padrão de parada de 13 unidades. Ler o padrão de parada da esquerda para a direita é o símbolo de parada (seguido por uma barra de 2 unidades) e ler o padrão de parada da direita para a esquerda é o símbolo de parada reversa (seguido por uma barra de 2 unidades).

    Verifique o cálculo do dígito [ editar ]

    O dígito de verificação é uma soma de verificação de módulo-103 ponderada. É calculado somando o 'valor' do código inicial aos produtos do 'valor' de cada símbolo multiplicado por sua posição na sequência do código de barras. O símbolo inicial e o primeiro símbolo codificado estão na posição 1. A soma dos produtos é então reduzida módulo 103. O restante é então convertido de volta para um dos 103 símbolos não delimitadores (seguindo as instruções fornecidas abaixo) e anexado ao código de barras , imediatamente antes do símbolo de parada.

    Para fins de cálculo do símbolo de verificação, os símbolos de mudança e troca de código são tratados da mesma forma que qualquer outro símbolo no código de barras. A soma de verificação é calculada sobre os valores dos símbolos, independentemente de qual conjunto de códigos está ativo no momento. Por exemplo, o valor "33" do conjunto de códigos C e o valor "A" do conjunto de códigos B são ambos considerados um valor de Código 128 de 33, e o dígito de verificação é calculado com base no valor de 33 vezes a posição do símbolo dentro do código de barras .

    Usando FNC4 para codificar caracteres altos (160–255) [ editar ]

    O símbolo especial FNC4 ("Função 4"), presente apenas nos conjuntos de códigos A e B, pode ser usado para codificar todos os caracteres Latin-1 ( ISO-8859-1 ) em um código de barras Code 128. [3] O recurso não é amplamente suportado e não é usado no GS1-128. [4] [5] Quando um único FNC4 está presente em uma string, o símbolo a seguir é convertido em ASCII como de costume e, em seguida, 128 é adicionado ao valor ASCII. (Se o símbolo a seguir for uma mudança, um segundo símbolo será usado para obter o caractere.) Se dois FNC4s forem usados ​​consecutivamente, todos os caracteres a seguir serão tratados como tal, até o final da string ou outro par de FNC4s . Entre os FNC4s duplos, um único FNC4 será usado para indicar que o seguinte caractere será ASCII padrão .[6]

    Larguras do código de barras [ editar ]

    Code128 especifica uma combinação de 6 barras alternadas e espaços (3 de cada) para cada símbolo. Assim, cada símbolo começa com uma barra e termina com um espaço. Em fontes de código de barras, a barra final geralmente é combinada com o símbolo de parada para criar um padrão de parada mais amplo. A tabela a seguir detalha as larguras associadas a cada barra e espaço para cada símbolo. A largura de cada barra ou espaço pode ser de 1, 2, 3 ou 4 unidades (módulos). Usando o exemplo acima, um 'A' seria representado com o padrão 10100011000, ou como larguras 111323 nas tabelas abaixo.

    O valor das larguras é derivado contando o comprimento de cada execução de 1 e 0 no padrão, começando da esquerda. Sempre haverá 6 execuções e os comprimentos dessas 6 execuções formam o valor de Larguras. Por exemplo, usando o padrão 10100011000, os comprimentos de execução são 1 (dígito 1), 1 (dígito 0), 1 (dígito 1), 3 (dígito 0), 2 (dígito 1), 3 (dígito 0). Relatar apenas os comprimentos de cada execução fornece 1, 1, 1, 3, 2, 3, produzindo assim um valor de largura de 111323.

    Os símbolos "Código A", "Código B" e "Código C" fazem com que todos os símbolos a seguir sejam interpretados de acordo com o subcódigo correspondente (ou seja, 128A, 128B ou 128C). O símbolo "Shift" alterna a interpretação de um único símbolo seguinte entre os subcódigos A e B.

    O caractere ASCII codificado depende da fonte do código de barras usada. Especialmente o caractere ASCII de valor 0 e de valor 95 e acima pode ser definido de forma diferente na fonte instalada.

    Os códigos FNC x são usados ​​para fins especiais. FNC1 no início de um código de barras indica um código de barras GS1-128 que começa com um identificador de aplicativo de 2, 3 ou 4 dígitos atribuído pelo Uniform Code Council , que explica os dígitos a seguir. Por exemplo, o identificador de aplicativo 421 indica que segue um código de país numérico ISO 3166-1 e um código postal de destino. Assim, o código postal dos EUA para a Casa Branca geralmente seria impresso como "(421) 840 20500", mas na verdade seria codificado como " [Start C] [FNC1] 42 18 40 20 50 [Code A] 16 [Check symbol 92] [Stop]"

    FNC2 (message append) can appear anywhere within a barcode to indicate that the barcode reader should store the current string and prepend it to the string of the next barcode that's read. It is not used by GS1-128.

    FNC3 (initialize) can appear anywhere within a barcode to instruct the barcode reader to initialize or reprogram itself according to the barcode string's instructions.

    FNC4 is used to represent an extended ASCII character set (see § Using FNC4 to encode high (160–255) characters). It is not used by GS1-128.[7]

    Availability[edit]

    For the end user, Code 128 barcodes may be generated by either an outside application to create an image of the barcode, or by a font-based barcode solution. Either solution requires the use of an application or an application add in to calculate the check digit and create the barcode.

    Barcode length optimization[edit]

    Code set C uses one code symbol to represent two digits, so when the text contains just digits it will generally result in shorter barcodes. However, when the string contains only a few digits or it's mixed with non-digit character, it does not always produce a more compact code than code sets A or B. Using code set C saves one symbol per two digits, but costs a mode-shift symbol to enter and exit the set. Thus, it is only worth using if there are enough consecutive digits. For example, encoding the string "X00Y" with code set A or B requires 7 code symbols ([Start B] 56 16 16 57 [checksum] [Stop]), while using code set C for the "X00Y" would result in a code 8 symbols long ([Start B] 56 [Code C] 00 [Code B] 57 [checksum] [Stop]).

    Using code set C is only advantageous under the following conditions:

    Location of digits Number of consecutive digits
    beginning of data 4+
    end of data 4+
    middle of data (surrounded by symbols from code set A or B) 6+
    entire data either 2 or 4+ (but not 3)

    At the end of a string, delaying the transition to code set C until there are an even number of digits remaining avoids an extra symbol. Consider the string "...01234": a delayed switch produces ... 0 [Code C] 12 34 [checksum] [Stop] but an early switch produces ... [Code C] 01 23 [Code A] 4 [checksum] [Stop].[8]

    For example, given the string "098x1234567y23", savings on barcode length using code set C are achieved only if it is applied to middle part of the string. For the beginning and ending part of the string, switching to code set C is not effective. As there are an odd number of digits in the middle of the string, the odd one must be use a different code, set, but it makes no difference whether this is the first or last; 16 symbols are required in either case: [Start B] 0 9 8 x 1 [Code C] 23 45 67 [Code B] y 2 3 [checksum] [Stop], or [Start B] 0 9 8 x [Code C] 12 34 56 [Code B] 7 y 2 3 [checksum] [Stop].

    Optimizing the length of the resulting barcode is important when barcode readers are used which must detect the entire barcode image at once in order to read it, such as common laser scanners. The longer the barcode is, the greater distance of laser barcode reader from barcode image is needed, making reading difficult or impossible above some threshold lengths/distances.

    The optimal encoding can be found using a dynamic programming algorithm.[9]
